> **ステータス**: ドラフト
> **最終更新**: 2026-04-14

> **対象範囲**: 音声アナウンス（TTS）のテンプレート・API・再生・設定

## 1. 概要

### 1.1 目的

試合の開始・勝者確定時に、選手名・所属・ルール等を音声で自動アナウンスする。OpenAI TTS を使用し、テンプレートベースでカスタマイズ可能。

### 1.2 対象ファイル

| ファイル                                    | 役割                                              |
| ------------------------------------------- | ------------------------------------------------- |
| `lib/speech.ts`                             | テンプレート定義・変数置換・TTS呼び出し・音声再生 |
| `app/api/tts/route.ts`                      | OpenAI TTS API のプロキシ                         |
| `app/api/public/announce-settings/route.ts` | アナウンステンプレートの公開読み取りAPI           |

| COURT_SPEC.md       | アナウンストリガー（試合開始・勝者設定） |

## 2. TTS API

### 2.1 エンドポイント

```
POST /api/tts
```

### 2.2 TTS サービス

OpenAI TTS-1（モデル: `tts-1`）

### 2.3 リクエスト

```json
{
  "text": "アナウンス文（必須）",
  "voice": "nova",
  "speed": 1.0
}
```

### 2.4 音声オプション

| 値        | 説明                     |
| --------- | ------------------------ |
| `nova`    | 女性・明瞭（デフォルト） |
| `shimmer` | 女性・柔らか             |
| `alloy`   | ニュートラル             |
| `echo`    | 男性・軽い               |
| `fable`   | 男性・ナレーション風     |
| `onyx`    | 男性・重厚               |

### 2.5 速度

- API 受付範囲: 0.25x 〜 4.0x
- UI スライダー範囲: 0.5x 〜 1.5x（step: 0.05）
- デフォルト: 1.0x

### 2.6 バリデーション

- `text` 未指定 → 400 エラー
- `voice` が許可リスト外 → `nova` にフォールバック
- `speed` が範囲外 or NaN → `1.0` にフォールバック

### 2.7 レスポンス

audio/mpeg バイナリ（音声ファイル）

## 3. テンプレートシステム

### 3.1 デフォルトテンプレート

**試合開始 (matchStart)**:

```
{{コート名}}、{{トーナメント名}}、{{試合ラベル}}。ルール、{{ルール}}。{{選手1流派＋道場}}、所属、{{選手1名前}}選手。対。{{選手2流派＋道場}}、所属、{{選手2名前}}選手。これより試合を開始します。
```

**勝者 (winner)**:

```
ただいまの試合は、{{勝者流派＋道場}}、所属、{{勝者名前}}選手の勝ちです。
```

### 3.2 テンプレート変数

**試合開始テンプレート (MATCH_VARS)**:
| 変数 | 説明 |
|------|------|
| `{{コート名}}` | コートの表示名（例: Aコート） |
| `{{トーナメント名}}` | トーナメント名（例: 男子一般部） |
| `{{試合ラベル}}` | 試合名/ラウンド名（例: 準決勝、第1試合） |
| `{{ルール}}` | ルール名 |
| `{{選手1名前}}` | 赤側の選手名（読みがな優先） |
| `{{選手1流派＋道場}}` | 赤側の所属（「流派、道場」形式） |
| `{{選手1流派}}` | 赤側の流派のみ |
| `{{選手1道場}}` | 赤側の道場のみ |
| `{{選手2名前}}` | 白側の選手名 |
| `{{選手2流派＋道場}}` | 白側の所属 |
| `{{選手2流派}}` | 白側の流派のみ |
| `{{選手2道場}}` | 白側の道場のみ |

**勝者テンプレート (WINNER_VARS)**:
| 変数 | 説明 |
|------|------|
| `{{勝者名前}}` | 勝者の選手名 |
| `{{勝者流派＋道場}}` | 勝者の所属 |
| `{{勝者流派}}` | 勝者の流派のみ |
| `{{勝者道場}}` | 勝者の道場のみ |

### 3.3 テンプレートカスタマイズ

- 管理画面のアナウンス設定タブで編集可能
- 「試合開始」「勝者発表」を `grid grid-cols-2` の均等割タブで切り替え（UIスタイルガイド準拠）
- サーバーサイド永続化: `/api/admin/settings` に `announce_templates` キーで保存（PUT、admin認証必要）
- 読み取り: `/api/public/announce-settings` から取得（認証不要。コート画面・タイマー画面等の非admin画面で使用）
- デフォルトにリセット可能
- プレビュー機能: サンプル変数でテンプレートを展開して表示

### 3.4 変数置換 (renderTemplate)

```typescript
renderTemplate(template: string, vars: Record<string, string>): string
```

- `{{変数名}}` パターンを対応する値に置換
- 未定義の変数は空文字列に置換

## 4. 所属名の整形

### 4.1 buildAffiliationForTts

所属文字列をTTS向けに整形:

```
"柔空会　本部道場" → "柔空会、本部道場"
```

- 全角スペースで分割し、読点（、）で結合
- 自然な読み上げのための区切りを挿入

### 4.2 splitAffiliationParts

所属文字列を流派と道場に分離:

```
"柔空会　本部道場" → { school: "柔空会", dojo: "本部道場" }
```

## 5. 試合ラベルの読み変換

### 5.1 normalizeMatchLabelForTts

漢字の試合ラベルをかな読みに変換:

| 入力     | 出力                   |
| -------- | ---------------------- |
| 決勝     | けっしょう             |
| 準決勝   | じゅんけっしょう       |
| 準々決勝 | じゅんじゅんけっしょう |
| 第1試合  | だいいちしあい         |
| 第2回戦  | だいにかいせん         |
| 1回戦    | いっかいせん           |

### 5.2 数字の読み (NUM_READING)

漢数字・全角数字・半角数字に対応。「1」→「いち」、「2」→「に」等のマッピング。

## 6. アナウンス関数

### 6.1 announceMatchStart

試合開始時のアナウンス。

**パラメータ**:

- `fighter1Name`, `fighter1Affiliation` — 赤側の表示名・所属
- `fighter2Name`, `fighter2Affiliation` — 白側の表示名・所属
- `roundLabel` — ラウンド名（決勝、第1回戦等）
- `fighter1NameReading?`, `fighter1AffiliationReading?` — 赤側の読みがな
- `fighter2NameReading?`, `fighter2AffiliationReading?` — 白側の読みがな
- `matchLabel?` — 試合番号ラベル（第1試合等）
- `rules?` — ルール名
- `templates?` — カスタムテンプレート
- `rulesReading?` — ルールの読みがな
- `courtName?` — コート表示名
- `tournamentName?` — トーナメント名

**処理**: `buildMatchStartText()` でテンプレートに変数を展開してテキストを組み立て → `speak()` で音声再生

### 6.1.1 buildMatchStartText

`announceMatchStart` と同じパラメータで、発話テキストのみを組み立てて返す（音声再生はしない）。TTS prefetch で利用。

### 6.1.2 prefetchTts

次の試合のアナウンステキストを事前に `/api/tts` に POST して音声を生成・キャッシュする。再生はしない。空文字列の場合は何もしない。fetch エラーは無視する。

### 6.2 announceWinner

勝者確定時のアナウンス。

**パラメータ**:

- `winnerName`, `winnerAffiliation` — 勝者の表示名・所属
- `nameReading?`, `affiliationReading?` — 読みがな
- `templates?` — カスタムテンプレート

### 6.3 announceCustom

任意テキストの読み上げ（テスト・プレビュー用）。

## 7. 音声再生

### 7.1 speak() 内部関数

```
1. getTtsSettings() で voice/speed を取得
2. POST /api/tts にテキスト・voice・speed を送信
3. レスポンスを Blob → Object URL に変換
4. new Audio(url) で Audio 要素を作成
5. onended ハンドラで Object URL を revoke
6. audio.play() を実行
```

### 7.2 エラーハンドリング

- `play()` の失敗は `console.error` でログのみ（アラートなし）
- TTS API エラーも同様（試合進行はブロックしない）

### 7.3 Audio 要素管理

- 毎回新しい Audio 要素を作成（再利用なし）
- Object URL は再生完了後に `revokeObjectURL()` で解放
- 同時再生防止: 再生中は新しい再生要求を無視（`speaking` フラグで排他制御）

### 7.4 再生中断 (stopSpeech)

```typescript
export function stopSpeech(): void;
```

- 再生中の Audio 要素を即座に停止（`audio.pause()`）し、Object URL を解放
- `speaking` フラグを `false` にリセット
- 再生中でない場合は何もしない

### 7.5 再生中オーバーレイ（操作画面）

アナウンス再生中（`isPlaying === true`）に操作画面全体を覆う半透明オーバーレイを表示し、誤操作を防止する。

**表示内容**:

- 半透明黒背景（`bg-black/70`）で全画面を覆う
- 画面中央に「アナウンス再生中」テキスト（大きめ）
- その下に「再生停止」ボタン（テキストの半分程度のサイズ）

**動作**:

- オーバーレイ表示中はキーボードショートカットを無効化（スペースキー含む）
- オーバーレイ表示中は背面のボタン・入力要素への操作が不可
- 「再生停止」ボタン押下 → `stopSpeech()` 呼び出し → 即座に再生停止 → `isPlaying = false` → オーバーレイ消去
- アナウンス再生が正常に完了した場合も `isPlaying = false` → オーバーレイ消去

## 8. ミュート制御

### 8.1 仕組み

試合ごとにミュート/アンミュートを切替可能。

### 8.2 永続化

- `localStorage` の `muted_match_ids` キー
- JSON 配列として保存
- ページリロードでも状態維持

### 8.3 動作

- ミュート中の試合: 試合開始・勝者アナウンスをスキップ
- 再アナウンスボタン（📢）はミュート中でも実行可能

## 9. TTS 設定

### 9.1 永続化

| localStorage キー | 内容     | デフォルト |
| ----------------- | -------- | ---------- |
| `tts_voice`       | 音声名   | `nova`     |
| `tts_speed`       | 再生速度 | `1.0`      |

### 9.2 設定 UI

管理画面のアナウンス設定タブ:

- **音声選択**: 6つのボタンから選択（2列グリッド）
- **速度スライダー**: 0.5x 〜 1.5x（step 0.05）、「遅い」「標準」「速い」ラベル
- **試し聞きボタン**: サンプルテキストで音声プレビュー
- **保存ボタン**: localStorage に永続化

### 9.3 サンプルテキスト

```
Aコート、男子一般部、準決勝。極真会所属、山田太郎選手。対。正道会館所属、鈴木一郎選手。これより試合を開始します。
```

## オフライン対応

詳細は [OFFLINE_SPEC.md](OFFLINE_SPEC.md) を参照。

TTS 音声の Cache API プリキャッシュ（prefetchTts で事前生成・保存）。オフラインではキャッシュ済み音声のみ再生可能。未キャッシュは無音。試合ごとにキャッシュをクリア。
