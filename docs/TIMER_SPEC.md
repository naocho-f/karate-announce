## 1. 概要

### 1.1 目的

試合進行をリアルタイムで管理・表示するスポーツタイマー機能。審判のコールに応じて操作者（タイムキーパー）が操作し、外付けモニターに試合状況を大画面表示する。

> **用語**: 本仕様書では「操作者」で統一する（= タイムキーパー）。

### 1.2 対象競技

- **Phase 1**: 総合武道（空手＋柔道）
- **Phase 2**: 柔道、空手単体、その他スポーツ

ルールプリセットにより競技ごとの設定を切り替えるため、コア実装は競技非依存。

### 1.3 設計方針

- **柔軟性最優先**: すべての数値・表示項目・動作をルールプリセットで設定可能にする
- **既存システム連携**: karate-announce のトーナメント・試合表データと双方向連携
- **デュアルモニター**: 1台の PC で表示ウィンドウと操作ウィンドウを分離（`BroadcastChannel` API）
- **アナウンス統合**: 既存のアナウンスシステム（コート画面）の操作もタイマー操作画面に統合

## 2. 画面構成

### 2.1 表示画面（`/timer/[courtId]`）

外付けモニターにフルスクリーン表示する画面。観客・選手・審判が見る。

- **アスペクト比**: 16:9（一般的な PC モニター）
- **操作要素なし**: 表示専用（タッチ・クリック不要）。ただし画面クリックで **Fullscreen API** によるフルスクリーン切り替えが可能（1クリックで enter、再クリックで exit）
- **自動更新**: `BroadcastChannel` 経由で操作画面からのメッセージをリアルタイム反映

### 2.2 操作画面（`/timer/[courtId]/control`）

操作者（タイムキーパー）が手元の PC 画面で操作する画面。**アナウンスシステムの操作も統合**し、この画面1つで試合進行のすべてを管理する。

- **操作方法**: マウス/タッチ操作 ＋ キーボードショートカット
- **タイマーのミニプレビュー**: 操作画面上部に表示画面の縮小版を常時表示。現在の状態をバッジで表示（例: 🟢 running、🟡 paused、🔴 time_up、⚪ idle）
- **操作パネル**: 大きく明確なボタン、押し間違い防止のレイアウト
- **ショートカット参照パネル**: 操作パネル横に常時表示。印刷用の専用ページ（`/timer/shortcuts`）も用意

### 2.3 通信方式

```
┌──────────────────┐    BroadcastChannel     ┌──────────────────┐
│   操作画面        │  ──────────────────→   │   表示画面        │
│  (メインモニター)  │    同一ブラウザ内       │  (外付けモニター)  │
│                    │    ラグ: ほぼゼロ       │  フルスクリーン    │
└──────────────────┘                         └──────────────────┘
```

- **チャンネル名**: `timer-{courtId}`
- **メッセージ形式**: タイマー状態全体を JSON で送信（差分ではなく全状態）。想定サイズ: 1〜2KB 程度（選手名・スコア・タイマー値・状態フラグ等。性能上の問題は生じない規模）
- **フォールバック**: 操作画面がタイマー状態を `localStorage` にも保存。表示画面は起動時に `localStorage` から初期状態を復元

### 2.4 画面の開閉・復旧・離脱防止

**開閉順序:**
| シナリオ | 挙動 |
|---------|------|
| 表示画面を先に開いた | `localStorage` から最後の状態を復元。なければ `idle` 画面を表示。操作画面が開かれたら即座に同期開始 |
| 操作画面をリロード | `localStorage` から状態復元。タイマーが `running` だった場合は **`paused` 状態で復元**（経過時間は保持、自動再開はしない）。操作者に「タイマーが一時停止されました」と通知。**理由**: リロード中にタイマーが無制御で進行することを防ぐ安全策。操作者が状況を確認してから手動で再開する |
| 操作画面を閉じた | 表示画面は最後の受信状態で静止。表示画面に「操作画面を開いてください」と案内表示 |

**localStorage 保存間隔:**

- タイマー動作中: **1秒ごと**にスナップショット保存
- タイマー停止中: 状態変更時のみ保存

**localStorage キー命名規則:** `timer-<用途>-{eventId}-{courtId}`（マルチテナント対応）
| キー | 用途 |
|------|------|
| `timer-state-{eventId}-{courtId}` | タイマー状態スナップショット（JSON） |
| `timer-active-{eventId}-{courtId}` | コート画面排他制御フラグ（値: `{timestamp: number}`。30秒TTL、10秒ハートビート） |

**操作画面の排他制御:**

- 同一コートの操作画面（`/timer/[courtId]/control`）は同一ブラウザ内で1タブのみ開くことを許可
- 2つ目のタブを開いた場合、`BroadcastChannel` で既存タブに問い合わせを送信。応答があれば「別タブで操作中です。このタブで操作を引き継ぎますか？」と確認ダイアログを表示
- 引き継ぐ場合: 既存タブにクローズ通知を送信し、既存タブは「別タブに操作が移動しました」と表示して操作を無効化。新タブが `localStorage` から状態を復元して操作を引き継ぐ
- 引き継がない場合: 新タブを自動的に表示画面にリダイレクト

**離脱防止:**

- 操作画面でタイマーが `idle` 以外の状態のとき、ページ離脱（タブを閉じる・リロード・URL遷移）を試みると `beforeunload` イベントでブラウザ標準の確認ダイアログを表示（メッセージはブラウザ依存、カスタマイズ不可）
- 表示画面には離脱防止なし（操作要素がないため）

## 3. ルールプリセット

### 3.1 概要

試合の全設定をまとめた「プリセット」を事前に作成・管理する。試合開始時にプリセットを選択すると、タイマーの動作・表示が自動で切り替わる。試合中もその場で微調整可能。

### 3.2 プリセット設定項目

> **注**: 各表の「例」列は設定可能な値の一例であり、デフォルト値ではありません。デフォルト値は Section 9.1 の DB 定義を参照してください。

#### 基本設定

| 項目             | 型          | 例                       | 説明                                                                     |
| ---------------- | ----------- | ------------------------ | ------------------------------------------------------------------------ |
| プリセット名     | string      | 「総合武道 ポイント制」  |                                                                          |
| 試合時間         | number (秒) | 120                      | メインタイマーの初期値。UIでは分:秒の分離入力                            |
| タイマー方向     | enum        | `countdown` / `countup`  | カウントダウン or カウントアップ                                         |
| 延長戦           | boolean     | true                     | 延長戦の有無                                                             |
| 延長方式         | enum        | `timed` / `sudden_death` | 時間延長（カウントダウン/アップ+再延長）or 先取延長                      |
| 延長時間         | number (秒) | 60                       | 延長戦の試合時間（`timed` の場合）                                       |
| 延長カウント方向 | enum        | `countdown` / `countup`  | 延長時のカウント方向（`timed` の場合）                                   |
| 最大延長回数     | number      | 0                        | 再延長の最大回数。0=無制限（`timed` の場合）                             |

#### レイアウト設定（行ベースエディタ）

フォントサイズは数値（vh単位）で自由に指定可能。上限なし（画面からはみ出すサイズも設定可）。

| 項目     | 型               | 説明                                                                     |
| -------- | ---------------- | ------------------------------------------------------------------------ |
| `layout` | jsonb (nullable) | `LayoutConfig` オブジェクト。null の場合は旧 enum フィールドから自動変換 |

**LayoutConfig 構造:**

```typescript
type LayoutConfig = {
  rows: LayoutRow[]; // 行の配列。並び順 = 表示順
  dividerThickness: number; // 区切り線の太さ (px)
  scoreGap: number; // 左右スコア間の隙間 (px)
  scoreItemGap: number; // スコア項目間（技あり・反則等）の間隔 (px)。デフォルト 8
  labelWazaari: string; // 技ありラベル（"W", "技あり", "技" 等）
  labelFoul: string; // 反則ラベル（"F", "反則", "反" 等）
  labelPoint: string; // ポイントラベル（"", "pt", "P" 等。空で非表示）
  labelNewaza: string; // 寝技ラベル（"寝技", "NEWAZA" 等）
};
type LayoutRow = {
  type: "timer" | "scores" | "player_names" | "match_info" | "newaza" | "spacer" | "timer_with_newaza";
  height: number; // vh。0 = flex-1（残りを均等分割）
  fontSize: number; // vh。制限なし
  align: "left" | "center" | "right";
  verticalAlign: "top" | "middle" | "bottom";
  subFontSize?: number; // scores用: 技あり・反則のフォントサイズ(vh)
  subAlign?: "left" | "center" | "right";
  timerRatio?: number; // timer_with_newaza用: メインタイマーの幅比率(0〜1、デフォルト0.75)
  scoreCenterMode?: "newaza" | "match_info"; // scores用: 中央に寝技 or 試合番号を表示
};
```

**行タイプ:**
| type | 説明 | 左右分割 |
|------|------|---------|
| `timer` | メインタイマー | なし |
| `scores` | ポイント・技あり・反則 | あり（赤/白） |
| `player_names` | 選手名 | あり（赤/白） |
| `match_info` | 試合番号・延長表示 | なし |
| `newaza` | 寝技タイマー | なし |
| `spacer` | 空白行 | なし |
| `timer_with_newaza` | メインタイマー＋寝技タイマー横並び | なし（内部で左右分割） |

**timer_with_newaza 行の詳細:**

- 左側（timerRatio、デフォルト75%）: メインタイマー（大フォント）
- 右側: 寝技タイマー2段表示（寝1/寝2）。寝技1回目・2回目をそれぞれ表示

**scores 行の scoreCenterMode:**

- `"newaza"`（デフォルト）: スコア行中央に寝技タイマーを表示
- `"match_info"`: スコア行中央に試合番号を大きく黄色で表示

#### テンプレート機能

プリセット新規作成時に、定義済みテンプレートから初期値を一括セットできる。テンプレート適用後は全フィールド自由に編集可能。

| テンプレート名 | 説明                                                                               |
| -------------- | ---------------------------------------------------------------------------------- |
| 空のプリセット | すべてデフォルト値から設定（従来の動作）                                           |
| 交流会         | タイマー+寝技横並び、スコア中央に試合番号表示（「A-1」形式）。赤=ピンク、白=グレー |

テンプレート定義は `lib/timer-templates.ts` で管理。

#### 注意（caution）機能

反則の前段階の警告。反則とは別にカウントし、表示色も異なる。

- `ScoreState.cautions: number` — 注意カウント
- 注意ボタン: 操作画面に黄色系ボタンとして表示（キーバインド: D=赤側、K=白側）
- 反則セル表示: 下から「注意 → 反則1 → 反則2 → 反則3」の4段
  - 注意セル: 点灯時は黄橙色（#E1D200）
  - 反則セル: 点灯時は選手色
  - 「注意から始まる場合」: 注意セルが黄橙色に点灯
  - 「いきなり反則の場合」: 注意セルは黒のまま、反則1が選手色で点灯
- 注意は自動判定（反則負け・相手ポイント付与）には影響しない
- Undo対応

**管理画面のレイアウトエディタ:**

- ドラッグ&ドロップで行の並べ替え
- 各行をクリックで展開 → 高さ・フォントサイズ（スライダー+数値入力）、配置（左/中/右、上/中/下）を設定
- 行の追加・削除
- 16:9 リアルタイムプレビュー（vh→px 変換で忠実に再現）

#### ブザー設定

メインタイマー・寝技タイマー・試合開始でそれぞれ異なるブザー音を設定可能。

| 項目                   | 型      | デフォルト          | 説明                                             |
| ---------------------- | ------- | ------------------- | ------------------------------------------------ |
| 試合開始ブザー         | enum    | `off`               | auto: 開始時に自動 / manual: 手動 / off: なし    |
| 試合終了ブザー         | enum    | `auto`              | auto: タイマー0で自動 / manual: 手動 / off: なし |
| 寝技タイムアップブザー | enum    | `auto`              | 同上                                             |
| 開始音源               | string  | `mid-square-single` | 内蔵音源ID（30種）or `custom`                    |
| 開始鳴動秒数           | numeric | 1.5                 | 0.5〜5.0秒                                       |
| 開始連続回数           | integer | 1                   | 1〜3回。音の間に0.3秒休止                        |
| メイン音源             | string  | `mid-square-single` | 内蔵音源ID（30種）or `custom`                    |
| メイン鳴動秒数         | numeric | 1.5                 | 0.5〜5.0秒                                       |
| メイン連続回数         | integer | 1                   | 1〜3回。音の間に0.3秒休止                        |
| 寝技音源               | string  | `mid-square-single` | 内蔵音源ID（30種）or `custom`                    |
| 寝技鳴動秒数           | numeric | 1.5                 | 0.5〜5.0秒                                       |
| 寝技連続回数           | integer | 1                   | 1〜3回                                           |
| カスタム音源パス       | string  | null                | アップロードした音源URL                          |

**内蔵音源一覧**（30種、Web Audio API でランタイム生成、ライセンスフリー）:

音程3段階（低音400Hz/中音800Hz/高音1200Hz）× 波形3種（矩形波/正弦波/ノコギリ波）× パターン3種（単音/二連/三連）= 27種 + 特殊3種（ホイッスル/ゴング/サイレン）

ID命名規則: `{pitch}-{wave}-{pattern}` (例: `mid-square-double` = 中音 矩形波 二連)

### 3.3 プリセット管理機能

- **CRUD**: 作成・編集・削除・一覧表示
- **複製**: 既存プリセットからコピーして新規作成
- **ルール紐付け**: 既存の `rules` テーブルのルールにプリセットを紐付け可能。トーナメント/試合にルールが設定されていれば、そのプリセットが自動適用される
- **管理画面**: `/admin` 内にタイマー管理ページを設置
- **カラー設定**: 6 つのカラーフィールド（`color_left`, `color_right`, `theme_bg_color`, `theme_timer_color`, `theme_timer_warn_color`, `theme_divider_color`）はネイティブカラーピッカー（`<input type="color">`）で選択。HEX コードも横に自動表示
- **レイアウトエディタ**: 行ベースのビジュアルエディタ。行の追加・削除・並べ替え（ドラッグ&ドロップ）、各行のフォントサイズ（vh数値、上限なし）・高さ・配置をスライダー+数値入力で自由に設定。16:9 リアルタイムプレビュー付き
- **試合中の設定変更**: 試合中（`running` / `paused`）に変更可能な項目は**表示設定・テーマ設定のみ**（色・フォントサイズ等）。ルール系設定（試合時間、ポイント先取り数、反則負け回数等）は `idle` / `ready` 状態でのみ変更可能。変更不可の項目はグレーアウト表示

## 4. タイマー動作仕様

### 4.1 メインタイマー

- カウントダウンの場合: 設定時間 → 0 で終了
- カウントアップの場合: 0 → 上限なし（手動停止 or ポイント先取り勝ち）
- **精度**: `requestAnimationFrame` + 開始時刻からの経過時間で計算（setInterval のドリフトを回避）
- **表示形式**: `M:SS`（例: `2:00`、`0:15`）。設定により `M:SS.s`（0.1秒単位）も可。カウントアップで10分以上は `MM:SS`（例: `12:34`）、100分以上は `MMM:SS`（実用上発生しない想定）
- **バックグラウンドタブ対応**: `requestAnimationFrame` はバックグラウンドで停止するが、経過時間は `Date.now()` ベースなので復帰時に正確な時間に追いつく

### 4.2 寝技タイマー

- メインタイマーと**同時進行**（メインは止まらない）
- 操作者が「寝技開始」→ 寝技タイマーがカウント開始。`newaza_direction` 設定に応じて `countup`（0 → 制限時間）または `countdown`（制限時間 → 0、残り時間表示）で動作
- 操作者が「寝技解除」→ 寝技タイマー停止
  - 経過時間が「無消費解除時間」以内 → 回数消費なし
  - 経過時間が「無消費解除時間」超 → 回数を1消費
- 寝技タイマーが制限時間に達した場合 → 自動停止 ＋ ブザー（設定による）。`newaza_stops_main` が true ならメインタイマーも自動停止（操作画面にトースト通知）。勝敗判定は主審が行う（タイマー側は停止のみ）
- **寝技解除時のメインタイマー連動**（`newaza_stops_main = true`）: 寝技を手動解除（Gキー）した場合、メインタイマーも `paused` に遷移。操作画面にトースト通知「寝技解除によりメインタイマーを停止しました」を表示
- **メインタイマーが `time_up` になった場合** → 寝技タイマーも即停止。経過が「無消費解除時間」以内なら回数消費なし（通常ルール適用）。ブザーはメインの `time_up` ブザーのみ鳴動（寝技タイムアップブザーは鳴らない）
- 回数制限に達した場合 → 寝技タイマー起動不可（操作ボタン無効化）
- **残り回数表示**: `newaza_limit_type === "limited"` の場合、操作画面の寝技ボタン下に「残りN回」を表示（`newaza_max_count - usedCount`）
- 寝技タイマーが制限時間で自動停止した後 → `G` キーで「解除」扱い（タイマーリセット、回数は自動停止時に消費済み）。再度 `G` キーで新たに寝技開始可能（回数制限内の場合）

#### 4.2.1 累積モード（`newaza_accumulate = true`）

- 寝技を解除しても**経過時間を保持**する。再開��時は前回の続きか���カウント
- 例: 制限時間2分 → 1回目30秒使用 → 解除（残り1:30保持） → 2回目開始（1:30からカウントダウン）
- **タイムアップ**: ブザー通知のみ。`newaza_stops_main` が true ならメインタイマーも自動停止（操作画面にトースト通知）。以降の寝技開始は不可（`exhausted` 状態）
- **回数���限と���併用**: 時間と回数のどちらか先に到達した方で制限（例: 2分以���かつ3回まで）
- **無消費解除時間**: 累積合計ではなく**今回区間の経過時間**で判定する
- **操作画面��示**: 寝技非アクティブ時も残り時間を常時表示。`exhausted` 時はボタン無効化 + 「制限時間到達」表示
- **表示画面**: 非アクティブ時も残り時間が表示される（既存の `showNewaza` 条��� `elapsedMs > 0` でカバー）

### 4.3 自動判定ルール

以下の条件が満たされた場合、自動で `finished` 状態に遷移する:

- **ポイント先取り勝ち**: ポイントが `point_win_threshold` に到達（反則による相手ポイント付与で到達した場合も含む）
- **反則負け**: 反則回数が `foul_loss_count` に到達
- **一本**: `ippon_wins` が true の場合
- **合わせ一本**: `combined_ippon_wins` が true かつ技あり2回以上の場合
- **同時成立**: 反則負けとポイント先取り勝ちが同一ボタン押下で同時に成立した場合、`foul_vs_point_priority` 設定に従う。具体例: 赤の反則ボタン押下 → 赤の反則カウントが `foul_loss_count` に到達（反則負け）＋ 同時に白へのポイント付与で白が `point_win_threshold` に到達（ポイント先取り勝ち）
  - `foul_priority`（デフォルト）: 反則負け優先。勝者 = 白、`result_method = "foul"`
  - `point_priority`: ポイント先取り優先。勝者 = 白、`result_method = "point"`
  - いずれの場合も勝者は同じ（反則した側の対戦相手）。記録される勝利方法のみ異なる
- 自動遷移後もアナウンスは自動再生されない（操作者が手動で再生）
- 誤操作は Undo（`Esc`）で取り消し可能

### 4.4 途中棄権・負傷

- 試合中（`running` / `paused`）に操作者が手動で試合を終了させるケース
- 操作画面の「試合結果」セクションで勝者を選択し、勝利方法で `棄権勝ち` or `負傷勝ち` を選択して確定
- 確定操作でタイマーは即停止し、`finished` 状態に遷移
- `time_up` を経由せずに `finished` に遷移する唯一のケース（自動判定を除く）

### 4.5 時間調整

- **±10秒 / ±1秒ボタン**: メインタイマーに即座に加減算（`paused` / `time_up` 時のみ表示）
- **調整ログ**: 調整が行われた場合、操作ログに記録（不正防止）

### 4.6 試合の流れ（ステートマシン）

```
[idle] ──(試合セット)──→ [ready/アナウンス選択]
                            │
                    (アナウンス再生 or スキップ)
                            │
                            ▼
                         [ready]
                            │
                        (開始ボタン)
                            │
                            ▼
                      [running] ←──(再開)── [paused]
                            │                   ▲
                        (待てボタン)─────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
       [time_up]      (一本/KO/先取り) (反則負け)
            │               │               │
     ┌──────┴──────┐        ▼               ▼
     ▼             ▼    [finished] ←────────┘
                             ▲
                             │(棄権/負傷 ← running/paused から直接)
  [extension]  [finished]     ▲
  (延長戦)         ▲          │
     │             │       (判定/引分)
     ▼             │          │
  [running]        └──────[time_up]
  (サドンデス)
     │(先取り)
     ▼
  [finished] ──(次の試合)──→ [idle]
```

**状態と図の対応:**

- `[idle]` = 待機（初期状態・試合間）
- `[ready]` = 準備完了（選手・ルール表示済み、音声プリフェッチ中）
- `[running]` = 試合中（延長戦含む）
- `[paused]` = 一時停止（「待て」）
- `[time_up]` = 時間切れ（判定入力・延長選択を行う状態）
- `[extension]` = 延長戦の準備状態（タイマーが延長用初期値に自動リセットされ、開始ボタン待ち）
- `[finished]` = 決着確定（結果書き戻し・次の試合セットを行う状態）

### 4.7 状態一覧

| 状態        | メインタイマー           | 操作可能                                             | 説明                                                                                                                                   |
| ----------- | ------------------------ | ---------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `idle`      | 停止                     | 試合セットのみ                                       | 初期状態・試合間                                                                                                                       |
| `ready`     | 停止（初期値表示）       | 開始                                                 | 試合選択直後はアナウンス選択画面を表示（「開始アナウンスを再生」or「アナウンスなしで試合準備画面へ」）。選択後に開始ボタンが表示される |
| `running`   | 動作中                   | 一時停止、ポイント、反則、寝技                       | 試合進行中（延長戦中も同じ状態）                                                                                                       |
| `paused`    | 一時停止                 | 再開、時間調整                                       | 「待て」状態                                                                                                                           |
| `time_up`   | 停止（0:00）             | 判定入力、延長開始、引き分け                         | カウントダウン終了。ここから延長 or 判定 or 引き分けを選択                                                                             |
| `extension` | 停止（延長用初期値表示） | 開始                                                 | 延長戦の準備状態。遷移した瞬間にタイマーが延長用初期値へ**自動リセット**される。開始ボタンで `running` へ遷移                          |
| `finished`  | 停止                     | 結果書き戻し、アナウンス再生、次の試合（確定後のみ） | 決着確定。結果確定（`resultWritten=true`）後に「次の試合へ」ボタンが表示され、`idle` に戻る                                            |

### 4.8 延長戦

#### 時間延長（`extension_mode = "timed"`）

- スコア引き継ぎ（リセットしない）
- `extension_timer_direction` に応じてカウントダウンまたはカウントアップ
- 延長時間（`extension_duration`）でタイマーをセット
- 時間切れで `time_up` に遷移。`extension_max_count`（0=無制限）の範囲内であれば再延長可能
- 再延長時は同じ延長時間で再度タイマーをリセットして開始

#### 先取延長（`extension_mode = "sudden_death"`）

- **スコアを 0-0 にリセット**して再開
- `extension_show_timer = true` の場合: カウントアップでタイマー表示
- `extension_show_timer = false` の場合: タイマー非表示（先取勝負のみ）
- 先にポイント/技あり/一本を取った方が勝利

#### サドンデスの自動判定

以下のいずれかが発生した瞬間、自動で `finished` 状態に遷移:

- ポイント加算により片方が相手を上回った（＝ポイント差が 1 以上になった瞬間）
- 技あり
- 一本
- 反則負け（`foul_loss_count` 到達）
- 誤操作の場合は Undo（`Esc`）で取り消し可能

#### 共通

- `finished` 遷移後も**勝者アナウンスは自動再生されない**
- 延長後も決着しない場合（再延長上限到達 or 先取延長後）、`time_up` 状態に戻り「判定」または「引き分け」を選択（`allow_draw` が true の場合のみ引き分け選択可能）

### 6.4 Undo（操作取り消し）

#### 対象操作

- ポイント加算 / 減算
- 反則加算
- 技あり
- 一本

#### 対象外

- タイマー操作（開始 / 停止 / リセット）
- 寝技の開始 / 解除（時間経過を伴う操作のため。回数消費も Undo 対象外）
- 結果確定（`finished` 状態への手動遷移）

#### 寝技回数の手動調整

- 寝技回数を誤った場合に備え、操作画面のサブ操作エリアに **寝技回数 +1 / -1 ボタン**を配置
- 回数制限が `limited` の場合のみ表示
- この操作は Undo 対象（スコア操作と同じ扱い）

#### 動作仕様

- 操作履歴をスタックで管理し、`Esc` キーを押すたびに1つずつ戻る
- **Undo の単位**: 1回のボタン押下（またはキー押下）で発生した全変更をセットで巻き戻す
  - 技あり Undo → 技ありカウント **＋** `wazaari_points` 分のポイント加算をセットで戻す
  - 反則 Undo → 反則カウント **＋** 相手への付与ポイント（`foul_point_value` 分）をセットで戻す
  - ポイント手動加算 Undo → ポイント1点のみ戻す
- **連続 Undo 可能**（スタックが空になるまで何回でも取り消せる。空の状態で `Esc` を押しても何も起きない。スタック上限: 100 操作。超過時は古い操作から破棄）
- **Redo なし**（Undo 後に同じ操作を再度行えばよい）
- 自動判定で `finished` に遷移した場合（例：ポイント先取り勝ち）、Undo するとポイントが戻り、**`finished` 遷移直前の状態に復帰する**（`running` 中の先取り勝ち → `running` に復帰、`time_up` 中の判定操作 → `time_up` に復帰）

#### スタッククリア条件

- **試合リセット時**: 「次の試合をセット」操作で `idle` に遷移したタイミング。セット前に確認ダイアログを表示（「現在の試合データは破棄されます。よろしいですか？」）
- **結果確定時**: 結果確定操作（`finished` → 結果書き戻し）でクリア。ただし「結果取り消し」で `time_up` に戻った場合も、スタックは復元されない

#### 結果確定の取り消し

- `finished` 状態からの復帰は Undo ではなく、専用の **「結果取り消し」ボタン** で `time_up` に戻す（別操作）
- **配置**: 操作画面の「試合結果」セクション内、結果確定ボタンの下。`finished` 状態のときのみ表示
- **確認ダイアログ**: 押下時に「結果を取り消しますか？」の確認を表示
- **Undo スタック**: 取り消し後もクリア済みのため復元されない（スコアは `finished` 時点の値が残る）
- **DB 書き戻し後の取り消し**: 結果書き戻し（DB保存）済みでも取り消し可能。その場合:
  - DB の `matches` レコードを `winner_id = NULL`, `status = "ongoing"`, `result_method = NULL`, `result_detail = NULL` に更新
  - **訂正ラベル**: 再度結果を確定して書き戻す際、`result_detail` に `"corrected": true` フラグを付与。ブラケット表示・ライブ速報で結果行の横に「(訂正)」ラベルを小さく表示（`text-yellow-400 text-[8px]`）
  - 操作ログにも「結果取り消し」「再確定（訂正）」を記録

### 7.5 コート画面との同時操作ポリシー

タイマー操作画面とコート画面（`/court`）の両方から勝者設定・アナウンスが可能なため、排他制御を行う。

**基本方針**: タイマー画面が active なコートは、コート画面側を読み取り専用にする。タイマーを使わない場合はコート画面が従来通り全機能を使える。

**仕組み:**

1. タイマー操作画面を開くと、`localStorage` に `timer-active-{courtId} = true` を書き込む
2. コート画面はポーリング時にこのフラグを検出し、該当コートの勝者設定ボタン・アナウンスボタンを非表示にする。代わりに「タイマー画面で操作中」の表示を出す
3. タイマー操作画面を閉じると（`beforeunload` イベント）フラグを削除 → コート画面が通常操作に戻る
4. フラグの有効期限: 書き込みから30秒でタイムアウト（タイマー操作画面は10秒ごとにフラグを更新）。ブラウザクラッシュ時にフラグが残り続けることを防止。**TTL 判定はコート画面側が担当**: フラグの値にタイムスタンプを含め（`{timestamp: Date.now()}`）、コート画面がポーリング時に `Date.now() - timestamp > 30000` なら期限切れと判定してフラグを無視する

**結果として:**

- タイマーなし・アナウンスだけ使いたい場合 → コート画面をそのまま使う（従来通り）
- タイマーを使う場合 → タイマー操作画面に集約。コート画面は表示のみ
- アナウンスの二重再生も防止される

**`lib/speech.ts` のリファクタ内容:**
既存の `speak()` は「テキスト生成→API呼び出し→再生」が密結合しているため、プリフェッチに対応できない。以下の3段階に分離する:

   - 内部関数 `buildAffiliationForTts()`, `splitAffiliationParts()` も export 化
2. **音声取得関数を新設**:
   - `fetchTtsAudio(text: string): Promise<HTMLAudioElement>` — TTS API を叩いて `Audio` オブジェクトを返す（再生はしない）
3. **既存関数はそのまま維持**:
   - `announceMatchStart()` / `announceWinner()` は内部で `buildXxxText()` → `fetchTtsAudio()` → `audio.play()` を呼ぶだけにリファクタ
   - コート画面からの利用は一切変わらない（後方互換）

**プリフェッチ管理は新ファイル** `lib/timer-audio-prefetch.ts` で担当:

- 3パターンの `Audio` オブジェクトのライフサイクル管理（取得・保持・破棄）
- プリフェッチ状態（loading / ready / error）の管理
- 再生メソッド（ボタン押下で `audio.play()` を呼ぶだけ → ラグなし）

### 8.2 音声プリフェッチ

試合がセットされた（`ready` 状態になった）タイミングで、以下の3パターンの音声を TTS API で事前取得する。

| パターン     | 使用テンプレート                             | タイミング   |
| ------------ | -------------------------------------------- | ------------ |
| 試合紹介     | 既存 `matchStart` テンプレート               | 試合セット時 |
| 赤（左）勝利 | 既存 `winner` テンプレートに赤選手情報を適用 | 試合セット時 |
| 白（右）勝利 | 既存 `winner` テンプレートに白選手情報を適用 | 試合セット時 |

**プリフェッチの流れ:**

1. `ready` 状態遷移時に3パターンの TTS テキストを生成
2. 各テキストを `/api/tts` に POST し、音声 blob を取得
3. `Audio` オブジェクトとしてメモリ上にプリロード
4. 取得完了 → 操作画面にアナウンス可能ステータスを表示
5. 操作者がボタン（または `1` `2` `3` キー）で即再生（**ラグなし**）

- 外部スピーカー接続を想定。ブラウザの音声出力先設定に依存

**プリフェッチ失敗時の動作:**

- 操作画面のアナウンスステータスに「取得失敗」を赤字で表示
- 該当のアナウンスボタンはグレーアウト（押下不可）
- 「再取得」ボタンを表示し、手動でリトライ可能
- 失敗してもタイマー操作・試合進行には影響しない（アナウンスなしで進行可能）

### 8.3 次の試合の先行プリフェッチ

- 「次の試合」の情報が確定した時点で、次の試合分の音声も先行してプリフェッチ開始
- 試合切替時に即アナウンス可能な状態を目指す
- 操作画面下部に「音声準備中...」→「準備完了」のステータスを表示

### 8.4 自動再生（Phase 2 で検討）

- ブザー鳴動 → 数秒後に結果アナウンス自動再生

### 11.2 状態管理

- タイマー状態は操作画面の React state で管理
- `BroadcastChannel` で表示画面に全状態を送信。送信タイミング: **状態変更時に即送信**（ポイント操作、状態遷移等）＋ タイマー動作中は **`requestAnimationFrame` 毎フレーム送信**（表示画面側で間引き不要、受信時に即描画）
- `localStorage` にもスナップショットを定期保存（復旧用）
- サーバーサイドの状態管理は不要（ブラウザ内で完結）

### 11.3 タイマー精度

```typescript
// requestAnimationFrame + Date.now() ベースの高精度タイマー
const startTime = Date.now();
function tick() {
  const elapsed = Date.now() - startTime;
  const remaining = duration - elapsed;
  // ... 表示更新
  if (remaining > 0) requestAnimationFrame(tick);
}
```

### 11.4 ブザー音

- **デフォルト**: 会場ブザー音（`.mp3` ファイルを `public/sounds/buzzer.mp3` に配置）
- **フォールバック**: デフォルト音源が読み込めない場合、Web Audio API で合成ビープ音を生成して代替
- **カスタム**: Supabase Storage にアップロードした音源ファイル。制限: `.mp3` / `.wav` / `.ogg` のみ、最大 2MB。**読み込み失敗時はデフォルト音源に自動フォールバック**し、操作画面に「カスタム音源の読み込みに失敗しました。デフォルト音源を使用します」と警告表示
- **テナント音源ライブラリ**: カスタム音源はテナント単位で `tenant_custom_sounds` テーブルに管理。一度アップロードした音源は同テナント内の全プリセットから選択可能。プリセットエディタで「カスタム音源」選択時、既存の音源一覧から選ぶか新規アップロードできる
- **再生**: `Audio` API で再生。外部スピーカー接続時はブラウザの音声出力設定に依存
- **操作者への案内**: 初回アクセス時にブザー音のテスト再生ボタンを目立つ位置に配置。「試合前にブザー音が鳴ることを確認してください」のガイダンスメッセージを表示。音が鳴らない場合のトラブルシューティング（ブラウザの音声許可、出力デバイス確認）も案内
